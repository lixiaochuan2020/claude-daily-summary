# Daily Conversation Summary Skill

You are generating a **daily knowledge summary report** from the user's Claude Code conversation logs.

## Arguments

- `$0` — Date in `YYYY-MM-DD` format (optional, defaults to today)
- `$1` — Project name filter (optional, substring match)

## Step 1: Extract Conversations

Run the extraction script to get clean Q&A pairs from conversation logs:

```bash
python3 ${CLAUDE_SKILL_DIR}/../../../scripts/extract_conversations.py --date "${0:-$(date +%Y-%m-%d)}" ${1:+--project "$1"}
```

If the script outputs nothing or errors, inform the user that no conversations were found for that date.

## Step 2: Analyze and Summarize

Read through ALL the extracted Q&A pairs carefully. Your job is to identify **every piece of knowledge, information, and insight** the user learned. Be thorough — the user wants to retain this knowledge.

For each distinct topic or concept discussed:
1. Identify the core question or problem the user had
2. Summarize the answer/solution clearly and concisely
3. Extract key concepts, patterns, commands, or techniques
4. Note any important caveats or gotchas mentioned

## Step 3: Generate the Report

Write a Markdown file to `reports/{date}-summary.md` (relative to the project root) using this structure:

```markdown
# Daily Knowledge Summary — {date}

> Auto-generated by `/daily-summary` skill

## Overview
- **Date**: {date}
- **Sessions reviewed**: N
- **Projects**: [list of project names]
- **Topics covered**: N

---

## Topics

### 1. {Topic Name}
**What was asked**: Brief description of the user's question/problem
**What I learned**:
- Key point 1
- Key point 2
- Key point 3

**Practical details**:
[Include any specific commands, code patterns, configurations, or techniques that were discussed. Use code blocks where appropriate.]

### 2. {Topic Name}
...

(Continue for ALL topics discussed)

---

## Quick Review (Flashcards)

| # | Question | Answer |
|---|----------|--------|
| 1 | ... | ... |
| 2 | ... | ... |
| ... | ... | ... |

Generate at least one flashcard for every distinct piece of knowledge. Aim for 10-20+ flashcards for a full day of conversations.

---

## Key Takeaways

1. Most important insight from today
2. Second most important insight
3. ...

---

<details>
<summary>Raw Q&A Log (click to expand)</summary>

[Paste the full extracted Q&A output here for reference]

</details>
```

## Important Guidelines

- **Be comprehensive**: Cover ALL knowledge exchanged, not just the highlights. The user wants to remember everything.
- **Be specific**: Include exact commands, code snippets, file paths, and configuration details — not vague summaries.
- **Flashcards are key**: The Quick Review section is the most important part for retention. Make questions specific and answers concise.
- **Group intelligently**: Combine related Q&As under a single topic heading rather than listing them one-by-one.
- **Skip meta-conversation**: Don't summarize discussions about project setup, file creation, or Claude's own planning process unless they contain genuinely useful knowledge.
- **Preserve technical accuracy**: Don't paraphrase technical details — keep them precise.

## Step 4: Confirm

After writing the report, tell the user:
- Where the report was saved
- How many topics and flashcards were generated
- A brief 2-3 sentence summary of the day's key learnings
